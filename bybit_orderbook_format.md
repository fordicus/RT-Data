* The file is in **NDJSON (newline-delimited JSON)** format.
* Parse line-by-line using `json.loads(line)` in Python.
* This format is ideal for **time-series processing of market microstructure data**.

## ðŸ“‚ Field Definitions

| Field     | Description                                                                        |
| --------- | ---------------------------------------------------------------------------------- |
| `topic`   | Topic name (e.g., `orderbook.200.UNIUSDC`)                                         |
| `ts`      | Timestamp (in milliseconds) generated by the system                                |
| `type`    | Type of message: `"snapshot"` or `"delta"`                                         |
| `data`    | Object that contains the full order book content                                   |
| â””â”€ `s`    | Symbol name (e.g., `UNIUSDC`)                                                      |
| â””â”€ `b`    | Bids. In snapshots, sorted by price descending                                     |
| â””â”€ `b[0]` | Bid price                                                                          |
| â””â”€ `b[1]` | Bid size. In deltas, size = `0` means full fill or cancel                          |
| â””â”€ `a`    | Asks. In snapshots, sorted by price ascending                                      |
| â””â”€ `a[0]` | Ask price                                                                          |
| â””â”€ `a[1]` | Ask size. In deltas, size = `0` means full fill or cancel                          |
| â””â”€ `u`    | Update ID. Sequential counter. `"u": 1` indicates full snapshot after system reset |
| â””â”€ `seq`  | Cross-sequence number. Lower `seq` indicates older data                            |
| `cts`     | Client-side timestamp when data was received                                       |
